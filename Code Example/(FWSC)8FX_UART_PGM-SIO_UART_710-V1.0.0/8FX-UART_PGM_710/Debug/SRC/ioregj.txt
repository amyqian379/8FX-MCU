はじめに

  MB95710シリーズ用サンプルI/Oレジスタファイルは、I/Oレジスタを
Ｃ言語の変数として宣言、および定義しているサンプルプログラムです。
  本書は、I/Oレジスタファイルのファイル構成、変数の宣言、および定義内容を説明し
ています。




I/Oレジスタファイルをご利用いただく場合の注意

  ・  I/Oレジスタファイルは、対象品種を限定して提供しています。したがって、他の
    品種に応用される場合には、提供している品種のプログラムを参考にして、該当す
    る品種のI/Oレジスタファイルを作成して下さい。

  ・  I/Oレジスタファイルを使用して、I/Oレジスタの操作を行うプログラムを記述す
    る場合、I/Oレジスタのアクセス順序、タイミング等をシステム的に考慮する必要が
    ありますので、ハードウェアマニュアルを参照して下さい。

  ・  I/Oレジスタファイルは、応用プログラムを開発する時に、役立てていただくため
    に作成した参考プログラムです。厳密には、個々のシステムに応じてプログラムに
    手を加える必要が出てくる場合もありますので、ご使用の際には十分評価の上ご使
    用下さい。

  ・  I/Oレジスタファイルの誤りについては、責任を負いかねる場合があり、また誤り
    が発見されても直ちに修正できない場合がありますのでご了承下さい。

第２章  I/Oレジスタファイル使用方法

  (1) 使用する品種を-cpuオプションに指定して、I/Oレジスタ定義ファイルをコン
      パイルして下さい。

        例： > fcc896s  -cpu mb***** -c _f2mc8fx.c

  (2) I/Oレジスタ変数を参照する応用プログラムを記述する場合、I/Oレジスタ宣言
      ファイルをインクルードして下さい。

        例：#include "_f2mc8fx.h"

            void func(){
                    .....
                IO_TCCS.bit.MODE = 1;
                    .....
            }

  (3) 使用する品種を-cpuオプションに指定して、応用プログラムをコンパイルして
      下さい。

        例： > fcc896s  -cpu mb***** -c sample.c

        注) CPUオプションで指定した品種が、I/Oレジスタファイルが使用可能な品種
            でない場合、コンパイル時に下記のエラーメッセージを出力します。

              #error "The I/O register file of the specified CPU option
                      does not exist"

  (4) 使用する品種を-cpuオプションに指定して、(1)で作成したオブジェクトファ
      イルと、(3)で作成した応用プログラムのオブジェクトファイルをリンクして
      下さい。

        例： > flnk896s -cpu mb***** -o sample.abs *.obj  -sc IO/IO=0x0000

        注) "-sc IO/IO=0x0000" の補足説明

            以下のWarningを抑止するために本設定を行って下さい。

            例： リンクしています...
                 *** W1373L: IO領域外への配置です (IO)

            Workbenchでは、以下のように設定して下さい。

            [プロジェクト(P)]-[プロジェクトの設定(J)]のリンカタグで、

            a)  カテゴリ(Y)の「配置/結合」で[セクション設定]をクリック
            b)  ROM/RAM領域名(R)を「アドレスで指定」に変更
            c)  セクション名(S)に「IO」を入力
            d)  アドレス(A)に「0x0000」を入力
            e)  内容種別(T)に「IO」を選択
            f)  [設定(E)]をクリック
            g)  [OK]をクリック

            [プロジェクトの設定(J)]の[OK]をクリックして終了

第３章  I/Oレジスタ宣言ファイルの宣言内容

  ハードウェアマニュアルに記載されているI/Oレジスタ名及びビット名に対応する、
I/Oレジスタシンボルを、以下のように宣言しています。

  ■ビット単位、又はバイト／ワード単位でアクセス可能なI/Oレジスタシンボルは、
    共用体型 (一部アクセスの都合上、構造体型) で宣言

  ■全ビットが書込み専用、またはビット単位でアクセスする意味がないI/Oレジスタ
    シンボルは、整数型で宣言

  ■I/Oレジスタシンボル名は、基本的に対応するI/Oレジスタ名の先頭に 'IO_'を付加
    した形で宣言（一部、Ｃ言語変数宣言上の問題や、アクセスの都合などで、複数の
    I/Oレジスタを１つのシンボル名で宣言したり、シンボル名を若干変更して宣言）

  ■ビットフィールドメンバ名は、対応するビット名で宣言
     (一部、アクセスの都合上、複数のビットを１つのメンバ名で宣言) 


  【例】I/OレジスタPDR0に対する宣言内容

    ・I/Oレジスタ構成

          7       6       5       4       3       2       1       0    bit No.
      ┏━━━┯━━━┯━━━┯━━━┯━━━┯━━━┯━━━┯━━━┓
      ┃  P07 │  P06 │  P05 │  P04 │  P03 │  P02 │  P01 │  P00 ┃PDR0
      ┗━━━┷━━━┷━━━┷━━━┷━━━┷━━━┷━━━┷━━━┛

    ・宣言内容

      #ifdef __IO_DEFINE
      #pragma segment IO=IO_PDR0, locate=0x0
      #endif

      typedef union {
                __BYTE  byte;
          struct {
               __BYTE  P00:1;
               __BYTE  P01:1;
               __BYTE  P02:1;
               __BYTE  P03:1;
               __BYTE  P04:1;
               __BYTE  P05:1;
               __BYTE  P06:1;
               __BYTE  P07:1;
          } bit;
      } PDR0STR;

      __IO_EXTERN PDR0STR    IO_PDR0;
      #define    _pdr0       IO_PDR0
      #define    PDR0        IO_PDR0.byte
      #define    PDR0_P00    IO_PDR0.bit.P00
      #define    PDR0_P01    IO_PDR0.bit.P01
      #define    PDR0_P02    IO_PDR0.bit.P02
      #define    PDR0_P03    IO_PDR0.bit.P03
      #define    PDR0_P04    IO_PDR0.bit.P04
      #define    PDR0_P05    IO_PDR0.bit.P05
      #define    PDR0_P06    IO_PDR0.bit.P06
      #define    PDR0_P07    IO_PDR0.bit.P07
